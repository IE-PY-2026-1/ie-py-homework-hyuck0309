# 🐍 [2026-01] 나만의 파이썬 소프트웨어 개발 프로젝트

## 1. 시나리오 제목
* 작업자 생산성 평가 시스템
<br>


## 2. 시나리오 (5~10줄)
* 이 프로그램은 작업자의 업무 수행량과 작업 시간을 입력받아 생산성을 계산하고 평가하는 시스템입니다.
* 사용자는 프로그램을 실행하면 먼저 작업자의 이름, 작업량, 작업 시간과 같은 기본 정보를 입력하게 됩니다. 입력된 데이터를 바탕으로 작업자의 생산성을 계산합니다. 생산성은 일정 시간 동안 수행한 작업량을 의미하며, 산업경영공학에서 작업 효율을 판단하는 핵심 지표입니다. 프로그램은 계산된 생산성 값을 기준으로 작업자의 효율 수준을 평가하고, 결과를 등급 형태(우수, 보통, 개선 필요 등)로 출력하여 사용자가 쉽게 이해할 수 있도록 합니다. 생산성이 높은 작업자는 우수한 성과를 낸 것으로 판단하고 성과금을 부여하고, 생산성이 낮은 경우에는 개선이 필요하다는 안내(교육)를 제공합니다. 추후에는 한 명의 작업자뿐만 아니라 여러 작업자의 데이터를 함께 저장하여 비교 분석할 수 있도록 확장할 계획입니다. 이를 통해 평균 생산성 계산, 작업자 간 성과 비교, 가장 우수한 작업자(올해의 작업자) 선출 등 심화된 기능을 구현할 수 있습니다. 이렇게 이 프로그램의 사용자는 원하는 기능을 선택하여 모든 작업자들의 정보를 입력, 조회, 분석할 수 있고 이를 바탕으로 전체 생산성을 관리할 수 있습니다.
<br> 


## 3. 예상 기능 및 메뉴 (최소 5개)
1. 작업자 정보 등록
2. 생산성 계산 및 평가
3. 생산 등급 판정
4. 성과금 및 안내 제공
6. 평균, 전체 생산성 계산
7. 우수 작업자 선발

## 🟦 [1차 과제: V1.0] 시나리오 기획
    
### **🤖 AI 파트너십 과정**

 1. **내용 1 : 전공 기반 주제 선정**
    * **프롬프트 요약:** ""산업경영공학과와 관련 있으면서도 C언어 기초를 배우고 있는 수준에서 구현 가능한 참신한 파이썬 프로그램 주제를 추천해줘."
    * **적용 내용:** AI가 제안한 여러 아이디어 중 '작업자 생산성 평가 시스템'이 내가 관심있는 생산관리 분야와 밀접하면서도 내 수준에 구현 난이도가 적절하다고 판단하여 채택하였다. 단순한 계산 프로그램이 아니라, 산업경영공학의 핵심 개념인 생산성 분석을 반영할 수 있어 과제의 방향성이 더 뚜렷해지면서 더욱 흥미를 가지게 되었다.
      
 2. **내용 2 : 로드맵 도출**
    * **프롬프트 요약:** ""1차 과제에서는 기본 기능(입출력, 사칙 연산 등)만 구현하고, 이후 과제에서 확장 가능한 구조로 시나리오를 설계해줘.""
    * **적용 내용:** 프로그램을 작업자 정보 입력 -> 생산성 계산 -> 등급 평가 -> 데이터 도출 -> 데이터 관리의 흐름으로 설계하였다. 그 결과 1차 과제에서는 입력과 계산 중심으로 구현하고, 이후에는 리스트와 반복문을 활용해 다수 작업자를 관리하는 구조로 확장할 수 있도록 기획하였다.

## 🟩 [2차 과제: V1.0] 입출력 + 리스트 + 조건문 - 향후 작성 예정
### **✨2차 과제 내용:**
  * 이번 과제는 1차 과제에서 구현한 작업자 생산성 평가 시스템을 확장하여, 여러 작업자의 데이터를 입력받고 관리할 수 있는 구조로 발전시키는 것을 목표로 합니다.
    사용자로부터 작업자의 이름, 작업량, 작업 시간을 입력받아 리스트에 저장하고, 이를 기반으로 생산성을 계산하여 작업자의 효율을 평가합니다.
    특히 for 반복문을 활용하여 여러 작업자의 데이터를 입력받고, if-elif-else 조건문과 and 논리 연산자를 활용하여 생산성 등급과 특별 평가를 부여하는 구조를 구현합니다.
    이를 통해 단순 계산 프로그램에서 벗어나, 여러 작업자를 비교하고 분석할 수 있는 기초적인 데이터 관리 시스템을 구축합니다.
  * input()으로 작업자 이름, 작업량, 작업 시간을 입력받아 리스트에 저장
  * 생산성 공식 적용: 생산성 = 작업량 / 작업 시간
  * if-elif-else를 사용하여 생산성 등급 판정
  * and 연산자를 활용하여 우수 작업자 특별 평가 기능 구현
  * len(), sum(), max(), index(), sort() 등을 활용하여 데이터 관리 기능 추가
    
### **🤖 AI 파트너십 과정**
1. **내용 1: 1차 -> 2차 구조(합산 관리 시스템) 확장 설계**
    * **프롬프트 요약:** “1차 과제의 단일 작업자 생산성 계산을 여러 작업자 관리 시스템으로 확장하고 싶어."
    * **적용 내용:** 1차때 구상만 해두었던 여러 작업자의 데이터 관리를 for 반복문과 리스트를 활용하여 구체화하였습니다. 이를 통해 단순 계산 프로그램에서 데이터 기반    관리 시스템으로 발전시킬 수 있었습니다.

2. **내용 2: 조건문 기반 로직 개선 및 프롬포트 틀 구**
    * **프롬프트 요약:** "생산성 기준으로 등급을 나누고 추가 조건으로 특별 평가를 주는 로직을 만들고 싶어."
    * **적용 내용:** if-elif-else 구조를 활용하여 생산성 구간별 등급을 나누고, and 조건을 활용하여 일정 기준 이상일 경우 특별 평가 및 보상을 부여하는 로직을 구체화하였습니다.
    
### **🛠️ Troubleshooting & 기술 회고:**
1. **문제 1: 입력값 계산 오류 (TypeError)
    * **원인:** input()으로 받은 값은 문자열이라 바로 계산이 불가능함
    * **해결:** int(), float() 형변환 적용
2. **문제 2: 잘못된 입력값(음수 데이터) 처리**
    * **원인:** 작업량 또는 작업 시간이 0 이하일 경우 계산 불가.
    * **해결:** if 조건문으로 예외 처리 후 다시 입력받도록 구현.
3. **문제3 : 여러 작업자 데이터 관리 어려움
    * **원인:** 여러 데이터를 동시에 관리하는 구조 부족
    * **해결:** 리스트를 활용하여 작업자 정보 저장 및 관리
