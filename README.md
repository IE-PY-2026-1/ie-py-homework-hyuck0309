# 🐍 [2026-01] 나만의 파이썬 소프트웨어 개발 프로젝트

## 1. 시나리오 제목
* 작업자 생산성 평가 시스템
<br>


## 2. 시나리오 (5~10줄)
* 이 프로그램은 작업자의 업무 수행량과 작업 시간을 입력받아 생산성을 계산하고 평가하는 시스템입니다.
* 사용자는 프로그램을 실행하면 먼저 작업자의 이름, 작업량, 작업 시간과 같은 기본 정보를 입력하게 됩니다. 입력된 데이터를 바탕으로 작업자의 생산성을 계산합니다. 생산성은 일정 시간 동안 수행한 작업량을 의미하며, 산업경영공학에서 작업 효율을 판단하는 핵심 지표입니다. 프로그램은 계산된 생산성 값을 기준으로 작업자의 효율 수준을 평가하고, 결과를 등급 형태(우수, 보통, 개선 필요 등)로 출력하여 사용자가 쉽게 이해할 수 있도록 합니다. 생산성이 높은 작업자는 우수한 성과를 낸 것으로 판단하고 성과금을 부여하고, 생산성이 낮은 경우에는 개선이 필요하다는 안내(교육)를 제공합니다. 추후에는 한 명의 작업자뿐만 아니라 여러 작업자의 데이터를 함께 저장하여 비교 분석할 수 있도록 확장할 계획입니다. 이를 통해 평균 생산성 계산, 작업자 간 성과 비교, 가장 우수한 작업자(올해의 작업자) 선출 등 심화된 기능을 구현할 수 있습니다. 이렇게 이 프로그램의 사용자는 원하는 기능을 선택하여 모든 작업자들의 정보를 입력, 조회, 분석할 수 있고 이를 바탕으로 전체 생산성을 관리할 수 있습니다.
<br> 


## 3. 예상 기능 및 메뉴 (최소 5개)
(예시)
1. 작업자 정보 등록
2. 생산성 계산 및 평가
3. 생산 등급 판정
4. 성과금 및 안내 제공
6. 평균, 전체 생산성 계산
7. 우수 작업자 선발

## 🟦 [1차 과제: V1.0] 시나리오 기획
    
### **🤖 AI 파트너십 과정**
(예시)
 1. **내용 1 : 전공 기반 주제 선**
    * **프롬프트 요약:** ""산업경영공학과와 관련 있으면서도 C언어 기초를 배우고 있는 수준에서 구현 가능한 참신한 파이썬 프로그램 주제를 추천해줘."
    * **적용 내용:** AI가 제안한 여러 아이디어 중 '작업자 생산성 평가 시스템'이 내가 관심있는 생산관리 분야와 밀접하면서도 내 수준에 구현 난이도가 적절하다고 판단하여 채택하였다. 단순한 계산 프로그램이 아니라, 산업경영공학의 핵심 개념인 생산성 분석을 반영할 수 있어 과제의 방향성이 더 뚜렷해지면서 더욱 흥미를 가지게 되었다.
      
 2. **내용 2 : 로드맵 도출**
    * **프롬프트 요약:** ""1차 과제에서는 기본 기능(입출력, 사칙 연산 등)만 구현하고, 이후 과제에서 확장 가능한 구조로 시나리오를 설계해줘.""
    * **적용 내용:** 프로그램을 작업자 정보 입력 -> 생산성 계산 -> 등급 평가 -> 데이터 도출 -> 데이터 관리의 흐름으로 설계하였다. 그 결과 1차 과제에서는 입력과 계산 중심으로 구현하고, 이후에는 리스트와 반복문을 활용해 다수 작업자를 관리하는 구조로 확장할 수 있도록 기획하였다.

## 🟩 [2차 과제: V1.0] 입출력 + 리스트 + 조건문 - 향후 작성 예정
### **✨2차 과제 내용:**
(예시)
  * 이번 과제는 용사 등록 및 정밀 진단 시스템 구현으로, 사용자로부터 데이터를 입력받아 리스트에 저장하고, 가중치 연산을 통해 결과값을 도출하는 시스템을 구축하는 것입니다. 특히 `if-elif` 조건문과 `and/or` 논리 연산자를 활용하여 상황에 맞는 등급과 특별 칭호를 부여하는 지능형 로직을 완성해야 합니다.
  * input()으로 용사의 이름, 체력, 마력, 공격력을 입력받아 리스트에 저장.
  * 가중치 공식을 적용하여 종합 전투력 산출: (체력 * 0.7) + (마력 * 0.5) + (공격력 * 1.8).
  * if-elif-else를 사용하여 S등급부터 F등급까지 5단계 등급 판정 로직 추가.
  * and 연산자를 활용하여 '전설의 대마법사' 등 특별 칭호 부여 기능 구현.
    
### **🤖 AI 파트너십 과정**
(예시) 
1. **내용 1: 길드 매니저 컨셉 및 로직 도출**
    * **프롬프트 요약:** "내가 길드 매니저가 되어 용사를 관리하는 프로그램을 만들 거야. 1학년 수준에서 리스트와 조건문을 활용해 등급을 매기는 재미있는 로직을 짜줘."
    * **적용 내용:** AI가 제안한 '가중치 전투력 산출' 방식과 'S등급이면서 마력이 높으면 대마법사 칭호 부여'라는 복합 조건 아이디어를 실제 코드에 반영함.

2. **내용 2: 효율적인 리스트 활용법 자문**
    * **프롬프트 요약:** "파이썬 기초 문법 단계(리스트 → 조건문 → 반복문 → 함수/딕셔너리)에 맞춰, '용사 길드 관리자' 컨셉의 프로그램이 어떻게 발전할 수 있을지 4단계 기능 확장 로드맵을 제안해줘."
    * **적용 내용:** 단순 입출력을 넘어, **'기초 정보 입력 및 리스트 저장(1차) → 전투 등급 판독기 구현(2차) → 무한 메뉴 시스템 구축(3차) → 딕셔너리를 활용한 다중 캐릭터 관리(4차)'**로 이어지는 논리적인 확장 로드맵을 확립함.
    
### **🛠️ Troubleshooting & 기술 회고:**
(예시) 
1. **문제 1: 숫자를 입력했는데 계산이 안 되는 현상 (TypeError)**
    * **원인:** `input()`으로 받은 데이터는 기본적으로 '글자'라서 숫자와 곱셈 연산이 불가능했음.
    * **해결:** `int()`와 `float()` 함수를 사용하여 입력값을 숫자로 바꾸는 '형변환' 마법을 부려 해결함.
2. **문제 2: 조건문 뒤에 자꾸 생기는 빨간 줄 (SyntaxError)**
    * **원인:** `if`와 `elif` 조건식 끝에 마법의 기호인 `:`(콜론)을 자꾸 빠뜨려서 발생함.
    * **해결:** 파이썬의 모든 조건문과 반복문 끝에는 항상 땡땡(`:`)이 들어가야 한다는 규칙을 학습하고 수정함.
